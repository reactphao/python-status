# 目录规范

本目录结构参考 `docs/REFERENCES.md` 中的 GitHub 仓库，并结合当前学习目标做最小必要设计。

## 设计原则

- 前后端分离：便于独立开发与部署
- 文档集中：统一记录进度、决策与计划
- 脚本与测试独立：便于自动化与质量保障

## 目录结构（v1.0）

```
.
├── backend/          # FastAPI 后端
├── frontend/         # React 前端
├── docs/             # 文档与记录
├── scripts/          # 自动化脚本
└── tests/            # 测试
```

## 目录说明

- `backend/`：Python/FastAPI 项目代码
- `frontend/`：React 项目代码
- `docs/`：学习记录、决策、进度与资料来源
- `scripts/`：环境/数据/任务相关脚本
- `tests/`：后端与前端测试
