# 学习计划

本计划基于“AI 应用工程师”主线，并使用一周一阶段的节奏推进。

## 阶段目标（先行版）

- 1-4 周：补齐 Python 工程化与 API 开发基础，能搭建简单后端服务
- 5-8 周：掌握 RAG/工具调用/评估基础，完成可演示 demo
- 9-12 周：完成一个具备业务闭环的 ToB 项目（需求→实现→评估）

## 第 1 周（草案）

目标：补齐 Python 工程化基础，理解模型原理基础概念，完成一个小型脚本实践。

### 第 1 天

- Python 环境：venv、pip、requirements.txt
- 项目结构：`src/`、`tests/`、`scripts/`
- 练习：创建并运行第一个脚本
- 具体安排：`docs/DAY1.md`

### 第 2 天

- Python 基础复盘：函数、模块、异常处理
- 输入输出：文件读写、JSON 处理
- 练习：读取 JSON 并生成简单报告

### 第 3 天

- 工程化：类型提示、日志、基础测试
- 练习：为已有脚本加入日志与简单测试

### 第 4 天

- 模型原理基础：模型是什么、训练与推理、上下文窗口
- 重要概念：token、prompt、temperature、top_p
- 练习：用同一问题测试不同参数效果并记录

### 第 5 天

- 小项目实践：命令行小工具（文本整理/摘要/分类）
- 文档记录：输入、输出、评估与改进点
